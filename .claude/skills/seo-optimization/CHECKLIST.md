# Periospot Article SEO & LLM Optimization Checklist

Use this checklist for every article optimization. Each item contributes to both traditional SEO and LLM discoverability.

---

## Pre-Optimization Analysis

- [ ] **Identify primary query**: What question does this article answer?
- [ ] **Identify secondary queries**: What related questions might lead here?
- [ ] **Review competitors**: What do top-ranking articles offer?
- [ ] **Assess current state**: Score 1-10 for LLM selection probability

---

## Title & Meta (Critical for Both SEO and LLM)

### Title Tag
- [ ] Length: 50-60 characters
- [ ] Primary keyword within first 30 characters
- [ ] Compelling hook or benefit
- [ ] No keyword stuffing
- [ ] Brand (Periospot) at end if space allows

**Formula**: `[Primary Keyword]: [Benefit/Hook] | Periospot`

**Examples**:
- ✅ "Peri-implantitis Treatment: Evidence-Based Clinical Protocols"
- ✅ "Bone Grafting Techniques: Success Rates and Best Practices"
- ❌ "All About Peri-implantitis and What You Need to Know"
- ❌ "Bone Grafting Bone Graft Techniques for Dental Implants"

### Meta Description
- [ ] Length: 150-160 characters
- [ ] Includes primary keyword naturally
- [ ] Contains a clear value proposition
- [ ] Ends with CTA (Learn how, Discover, Get the protocols)
- [ ] Compelling enough to click

**Formula**: `[Direct answer snippet]. [Value prop]. [CTA].`

**Example**:
> "Peri-implantitis affects 12-43% of implants. Learn evidence-based treatment protocols, success rates, and prevention strategies from clinical experts."

---

## Opening Paragraph (Most Critical for LLM Selection)

- [ ] **First sentence**: Directly answers the primary query
- [ ] **Contains**: At least one specific number/statistic
- [ ] **Contains**: Expertise signal (clinical reference, author credential)
- [ ] **Standalone value**: Could be cited as complete answer
- [ ] **Length**: 50-100 words
- [ ] **No fluff**: No "In this article we will..." phrases

### Opening Paragraph Template

```markdown
[DIRECT ANSWER to primary query]. [STATISTIC or clinical fact].
[OVERVIEW of key points/treatments/methods]. [EXPERTISE SIGNAL -
reference to guidelines, research, or clinical experience].
```

---

## Content Structure

### Heading Hierarchy
- [ ] **H1**: One per page, contains primary keyword
- [ ] **H2s**: 4-8 major sections, question-based or topic-based
- [ ] **H3s**: Subsections under each H2 as needed
- [ ] **Logical flow**: Introduction → Body → Conclusion/Takeaways
- [ ] **No skipped levels**: H1 → H2 → H3 (never H1 → H3)

### Section Requirements
- [ ] Each H2 section starts with key takeaway
- [ ] Each section provides specific, actionable content
- [ ] Sections are 150-400 words each
- [ ] Natural transitions between sections

### Required Sections (Adapt to Topic)

For **Condition** articles (e.g., Peri-implantitis):
- [ ] Definition/Overview
- [ ] Causes/Etiology
- [ ] Signs and Symptoms
- [ ] Diagnosis
- [ ] Treatment Options
- [ ] Prevention/Maintenance
- [ ] FAQ
- [ ] Key Takeaways

For **Technique** articles (e.g., Bone Grafting):
- [ ] Overview/Indications
- [ ] Patient Selection
- [ ] Step-by-Step Protocol
- [ ] Materials/Equipment
- [ ] Post-operative Care
- [ ] Complications/Management
- [ ] Success Rates/Evidence
- [ ] FAQ
- [ ] Key Takeaways

For **Comparison** articles (e.g., PRF vs PRP):
- [ ] Overview of Both
- [ ] Key Differences (Table)
- [ ] Clinical Indications for Each
- [ ] Evidence Comparison
- [ ] Clinical Decision Guide
- [ ] FAQ
- [ ] Key Takeaways

---

## Expertise Signals

- [ ] **Author attribution**: Name + credentials visible
- [ ] **Clinical references**: Guidelines (AAP, EFP, ITI)
- [ ] **Research citations**: When discussing evidence
- [ ] **Specific numbers**: Success rates, timeframes, measurements
- [ ] **Clinical scenarios**: Real-world application examples
- [ ] **Professional terminology**: Used correctly, explained when needed

---

## Content Quality

### Specificity
- [ ] Includes specific numbers (percentages, timeframes, dosages)
- [ ] Names specific techniques/products when relevant
- [ ] Provides measurable outcomes
- [ ] Avoids vague language ("many patients," "often")

### Comprehensiveness
- [ ] Covers topic thoroughly (could be the only resource needed)
- [ ] Addresses common questions and concerns
- [ ] Includes edge cases and considerations
- [ ] Provides context for clinical decision-making

### Actionability
- [ ] Clear protocols or steps when applicable
- [ ] Specific recommendations
- [ ] Clinical tips and guidance
- [ ] Next steps for the reader

---

## Special Elements

### Blockquotes (for Key Clinical Guidance)
- [ ] Used for important clinical tips
- [ ] Used for guideline quotes
- [ ] Used for critical warnings

**Format**:
```markdown
> **Clinical Tip**: Always verify implant stability before loading.
> ISQ values >65 indicate adequate stability for immediate loading.
```

### Lists (for Protocols, Steps, Comparisons)
- [ ] Bulleted lists for non-sequential items
- [ ] Numbered lists for sequential steps
- [ ] Used for risk factors, indications, contraindications

### Tables (for Comparisons, Data)
- [ ] Used for comparing techniques/materials
- [ ] Used for presenting data clearly
- [ ] Include clear headers

### Images
- [ ] Relevant clinical images when available
- [ ] Descriptive alt text (see below)
- [ ] Captions explaining clinical significance

---

## Image Optimization

### Alt Text Requirements
- [ ] Describes what's in the image
- [ ] Includes clinical context
- [ ] 125 characters max
- [ ] No "image of" or "picture of"

**Examples**:
- ✅ "Peri-implantitis with 5mm bone loss visible on periapical radiograph"
- ✅ "Step 3 of socket shield technique showing buccal root fragment"
- ❌ "Image of dental implant"
- ❌ "peri-implantitis-treatment.jpg"

### Image Filename
- [ ] Descriptive, hyphenated
- [ ] Contains relevant keywords
- [ ] Example: `peri-implantitis-bone-loss-radiograph.jpg`

---

## Internal Linking

- [ ] **Minimum 3 internal links** to related Periospot articles
- [ ] Links are contextually relevant
- [ ] Anchor text is descriptive (not "click here")
- [ ] Links to both foundational and advanced content

**Good Internal Link Examples**:
```markdown
- Related: [Understanding Peri-implant Mucositis](/blog/peri-implant-mucositis)
- See also: [Implant Maintenance Protocols](/blog/implant-maintenance)
- Advanced: [Surgical Treatment of Peri-implantitis](/blog/surgical-peri-implantitis)
```

---

## FAQ Section

- [ ] **Included for every article**
- [ ] 3-5 questions minimum
- [ ] Questions match what users actually search
- [ ] Answers are direct and complete
- [ ] Each answer is 50-150 words

**FAQ Format**:
```markdown
## Frequently Asked Questions

### [Exact question users search]?

[Direct answer in first sentence]. [Supporting details]. [Specific numbers/timeframes if applicable].
```

---

## Closing Elements

### Key Takeaways Section
- [ ] 4-6 bullet points
- [ ] Summarizes main actionable points
- [ ] Can stand alone as quick reference

### Conclusion
- [ ] Reinforces main message
- [ ] Provides clear next steps
- [ ] Links to related resources

---

## Technical SEO

- [ ] **URL slug**: Short, keyword-rich, no dates
- [ ] **Schema markup**: Article type with author, date, organization
- [ ] **Canonical URL**: Set correctly
- [ ] **Mobile-friendly**: Readable on all devices
- [ ] **Page speed**: Images optimized, no heavy scripts

---

## Final Quality Check

### The LLM Selection Test
Ask yourself these questions:

1. [ ] **Would an LLM cite this first paragraph as a complete answer?**
2. [ ] **Is the expertise evident within 10 seconds of reading?**
3. [ ] **Could a dental professional learn something actionable here?**
4. [ ] **Is this better than the top Google result for this query?**
5. [ ] **Would I trust this source in my own clinical practice?**

If any answer is "no," identify and fix the gap.
