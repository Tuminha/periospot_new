# SEO Audit - Periospot (Next.js)

Date: 2026-01-04
Scope: `periospot-nextjs` app (current main frontend)

## Executive Summary
The core SEO foundation is in place (global metadata, JSON-LD for Organization/Website, per-post and per-product metadata with Yoast mappings, sitemap and robots), but several high-impact gaps remain:
- GA tracking is implemented but the Measurement ID is missing from environment files, so no data will flow.
- Language routes are linked in the header but do not exist, creating broken internal links.
- Several pages should be `noindex` (search, cart, checkout, auth, dashboard) but currently inherit indexable metadata.
- Open Graph assets referenced in metadata (`/og-image.jpg`) are missing from `public/`.
- Sitemap includes non-existent `/contact` and a hard-coded category list instead of data-driven categories.

## Critical Gaps (High Priority)

### 1) GA Measurement ID Not Set
- **Finding**: `NEXT_PUBLIC_GA_MEASUREMENT_ID` is not present in `.env` or `.env.local`.
- **Impact**: GA tracking and DebugView will not receive any events.
- **Fix**: Add `NEXT_PUBLIC_GA_MEASUREMENT_ID=G-4KQ5WD0232` to `.env.local` and the same variable in Vercel.

### 2) Broken Language Links in Header
- **Finding**: Header links point to `/blog/es`, `/blog/pt`, `/blog/zh`, `/library/es`, `/library/pt`, `/library/zh`, but those routes do not exist.
- **Impact**: Broken internal links + crawl errors.
- **Fix**: Create language routes or remove these links until localized content exists.
- **Location**: `periospot-nextjs/src/components/Header.tsx`

### 3) Indexable Utility Pages That Should Be Noindex
- **Finding**: Search, cart, checkout, login/signup, dashboard, admin pages inherit the global metadata and are indexable.
- **Impact**: Low-quality pages can appear in search and dilute crawl budget.
- **Fix**: Add page-level metadata with `robots: { index: false, follow: false }` for:
  - `/search` (`periospot-nextjs/src/app/search/page.tsx`)
  - `/cart` (`periospot-nextjs/src/app/cart/page.tsx`)
  - `/checkout` (`periospot-nextjs/src/app/checkout/page.tsx`)
  - `/login`, `/signup`, `/auth/*`, `/dashboard/*`, `/admin/*`

### 4) Missing OG Image Asset
- **Finding**: Global metadata references `/og-image.jpg` but the file is not in `public/`.
- **Impact**: Social share previews may fall back or fail.
- **Fix**: Add `public/og-image.jpg` and update `metadata.openGraph.images` if needed.
- **Location**: `periospot-nextjs/src/app/layout.tsx`

### 5) Sitemap Issues
- **Finding**: `/contact` included in sitemap but no page exists.
- **Finding**: Categories list is hard-coded and incomplete.
- **Impact**: Search engines see 404s or miss categories.
- **Fix**: Remove `/contact` unless a page is added. Generate category URLs from `getAllCategories()`.
- **Location**: `periospot-nextjs/src/app/sitemap.ts`

## Medium Priority Gaps

### 6) Missing Hreflang/Alternates for Localized Content
- **Finding**: Only root-level alternates in layout. Blog and library posts do not declare language alternates.
- **Impact**: Poor international targeting and possible duplicate content.
- **Fix**: Add `alternates.languages` in metadata for language-specific routes when content exists.

### 7) Search Results Page Lacks Canonical/Noindex
- **Finding**: `/search?q=...` can be indexed with duplicate parameterized URLs.
- **Fix**: Add `noindex` and optionally canonical to `/search`.

### 8) Missing Breadcrumb Structured Data
- **Finding**: Blog post pages have visible breadcrumbs but no `BreadcrumbList` JSON-LD.
- **Fix**: Add BreadcrumbList schema to `blog/[slug]` (and product pages).

### 9) Missing RSS/Atom Feed
- **Finding**: No feed for blog posts.
- **Impact**: Missed distribution channel and SEO signals.
- **Fix**: Add `/feed.xml` using `getAllPosts()`.

## Content/Asset Gaps

### 10) Potential Missing Alt Text in Legacy Content
- **Finding**: HTML from WordPress may contain images without alt attributes.
- **Fix**: Add a post-processing step to ensure alt tags exist or use title fallback in migration.

### 11) Placeholder Images in Library Page
- **Finding**: Library uses `/images/...` placeholders that are not in `public/`.
- **Impact**: Broken image URLs hurt UX and SEO.
- **Fix**: Add assets or remove placeholders.

## Recommendations (Action List)

1. Add GA Measurement ID to `.env.local` and Vercel.
2. Fix/implement language routes in header.
3. Add `noindex` metadata for non-content pages (search/cart/checkout/auth/dashboard/admin).
4. Add `public/og-image.jpg` and verify Open Graph previews.
5. Update `sitemap.ts` to use real categories + remove `/contact` until it exists.
6. Add BreadcrumbList JSON-LD for blog posts + products.
7. Add RSS/Atom feed for blog posts.
8. Audit legacy content images for missing alt attributes.
9. Align language alternates per page once translations exist.

## Notes
- Per-post and per-product SEO is already wired to Yoast fields in JSON, which is a strong foundation.
- Once content migration is complete, re-run a link/alt audit to validate real content and assets.
